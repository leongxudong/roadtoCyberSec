# Corrective Action Plan Template

This template can be used to track remediation actions arising from audits, risk assessments, incidents, vendor reviews, or compliance gaps.

## Finding Description

Describe the issue clearly:

- What was observed?
- What requirement, control, or expectation was not met?
- What evidence supports the finding?
- What is the business or security impact?

## Root Cause Categories

| Category | Description |
|---|---|
| Process Gap | Procedure is missing, unclear, or not followed |
| Control Gap | Control is missing or insufficient |
| Ownership Gap | Responsibility is unclear or not assigned |
| Awareness Gap | Users or teams are unaware of required practice |
| Technical Gap | System configuration, tooling, or technical control is inadequate |
| Vendor Gap | Vendor obligation, evidence, or service commitment is insufficient |

## Corrective Action Quality Check

A good corrective action should be:

- Specific
- Assigned to an owner
- Time-bound
- Proportionate to the risk
- Verifiable through evidence
- Capable of preventing or reducing recurrence

## Closure Evidence Examples

| Corrective Action Type | Possible Evidence |
|---|---|
| Policy update | Approved policy document, version history, communication record |
| Procedure update | Updated SOP, workflow diagram, training record |
| Technical control | Screenshot, configuration export, change ticket, test result |
| Vendor remediation | Vendor confirmation, revised contract clause, SLA update, assessment evidence |
| Awareness activity | Attendance record, communication, training material, completion report |

## Notes

Closure should be based on evidence, not verbal confirmation alone. If full remediation is not possible, document compensating controls and residual risk acceptance.
